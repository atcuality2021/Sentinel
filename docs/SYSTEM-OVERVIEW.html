🛡What Is Sentinel?

Sentinel is an autonomous research-to-action agent. You give it a target — a competitor, a client, a software product, a market, an academic topic, a destination — and it plans the work, runs a multi-agent research pipeline, synthesises a structured intelligence brief, and recommends prioritised next actions.

The core moat: private data fusion without leakage + zero-cloud-egress mode for regulated deployments. Every finding carries a boundary tag (PUBLIC vs PRIVATE) enforced structurally at the memory layer — it cannot be bypassed by prompt.

Two-Pass Sovereign Engine

Pass 1
Gemma-4 12B · Tool-Calling

Plans queries, calls web search, extracts per-source notes. Non-streamed to avoid 524 timeout. Handles all tool-use decisions.

Pass 2
Gemma-4 26B · Reasoning

Synthesises all extracted notes into the structured output schema. SSE-streamed to the browser. Handles all long-form reasoning.

The two models never run simultaneously — tool-calling and reasoning are architecturally separated. Under on_prem_required, no Gemini object is constructed.

🧠Memory System — Three Tiers, SM-2

LayerWhat It StoresBehaviour
Semantic (entity) Facts extracted from past runs SM-2 spaced-repetition: used facts strengthen; stale facts decay and drop
Episodic Run records — what was researched, when, findings summary Injected into new runs about the same entity; builds on prior context
Procedural Strategy playbooks (Markdown, editable) Loaded at runtime by the strategy agent step

User Feedback Loop (new)

Thumbs up (+1) reinforces SM-2 strength on entity facts — high-quality research stays in recall longer. Thumbs down (-1) weakens ease and strength — poor results decay faster. Feedback is persisted in SQLite and applied immediately via the FeedbackStore._apply_to_memory hook.

🔐Governance / Sovereignty

ModeMeaningWho Uses It
cloud_ok Gemini allowed everywhere Internal tools, prototypes, client-approved cloud
on_prem_preferred Default on-prem; Gemini only with an ADR Most BiltIQ / Sentinel deployments
on_prem_required Hard block — no Gemini object constructed, provably Healthcare, BFSI, Government, Defence, air-gapped

Search providers: Gemini grounding · DuckDuckGo · Brave · SerpAPI · Google CSE · SearXNG. block_cloud_on_private=True forces on-prem for any run touching private data, even in cloud_ok mode.
