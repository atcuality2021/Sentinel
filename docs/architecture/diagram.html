Sentinel — Sovereign Intelligence Agent · Architecture

Universal research-to-action platform on Google ADK 2.2 — built for the Google for Startups AI Agents Challenge (Track 1: Build, APAC). Project → Task → Plan → parallel DAG of domain skills, executed on a governed dual backend (cloud Gemini or on-prem dual-tier Gemma-4 sovereignty), with a structural public/private boundary, layered memory (semantic · episodic · KB · procedural) and config-driven external MCP toolsets. 4Tool layer — what pass-1 agents can touch

All tools are fail-soft. Search has a per-run call budget (8) with a soft "synthesize now" stop; DDG staggers against throttling. External MCP toolsets are config-driven, domain-scoped and cloud-gated — a sovereign run never builds one. Tool output passes the injection/PII sandbox before reaching a reasoner.

BPublic / private boundary — structural, not prompt-based

7End-to-end research run

  1. Operator submits an objective → GET /projects/{id}/plan; task reused or created; optional client_url starts a background KB crawl in parallel.
  2. plan_task: deterministic template if the domain shape is known, else LLM Plan-pass — typed Plan, id forced to plan-{task.id} (one plan per task).
  3. staff_plan stamps each step with an agent_spec_id from the registry (eval-score ranked).
  4. Task + plan persisted (status=planned) → plan review page (PRG).
  5. gate_proposal — autonomy setting decides: propose (human approves) or autonomous (straight through). Backend/search policy resolved by governance, honoring per-project compliance.
  6. Approve → POST …/tasks/{tid}/run (re-run selector: auto / gemini / vllm / vllm-26b) → _approve_and_run reloads the persisted plan and resets step statuses.
  7. run_dag pre-run injection: entity memory + episodic recall + KB top-5 (sandboxed) + prospective follow-ups + persona + user profile → memory_context.
  8. Level-parallel execution: per step — cache check → pass-1 12B with search/KB/MCP tools → optional map-reduce extract → sandbox → pass-2 26B SSE synthesis. Budgets degrade gracefully; failures never crash the DAG.
  9. assemble_generic → typed Result (artifacts · citations · degraded flags) + optional persona render and rubric grade.
  10. Persist: result on task · RunRecord · entity relations · KB auto-ingest of artifacts · procedural trace · skill curation · priority recompute.
  11. 303 → task page: DAG graph, step timeline, artifact HTML, feedback (±1 → SM-2), post-run chat, PDF/HTML export.
