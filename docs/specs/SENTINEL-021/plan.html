Step 1 — Pure resolution policy _pick_winner(a, b) M

Rationale: The arbitrary keep="a" is the root defect. Fix the decision first as a pure function, before wiring any trigger — it is the shared policy for both the write-time path and the curator, so it must exist and be proven first.

Files: src/sentinel/memory/store.py (new module-level _pick_winner); tests/test_memory_reconcile.py (new).

Logic: tie-break order — newer created_at → boundary trust (PRIVATE>PUBLIC) → higher strength then access_count → stable lexicographic id. No DB, no clock, no LLM.

Tests:
  • test_pick_winner_newer_wins — later created_at is winner (AC2)
  • test_pick_winner_private_beats_public_on_tie — equal created_at → PRIVATE wins (AC2/OQ-1)
  • test_pick_winner_strength_then_id_tiebreak — equal time+boundary → strength, then id (AC2)
  • test_pick_winner_order_independent — _pick_winner(a,b) and (b,a) return the same winner (AC2)

Gate: pytest tests/test_memory_reconcile.py -k pick_winner — GREEN

Step 2 — superseded_by column (migration + schema + row I/O) S

Rationale: The link is needed before auto-resolve can set it. Land it additive + dormant so a pre-migration DB is provably unaffected.

Files: store.py — add ("superseded_by","TEXT") to _MEMORY_MIGRATIONS (293); read via _opt_col in _row_to_entry (335); add to the _insert column list + VALUES (765). schema.py — superseded_by: str | None = None on MemoryEntry (91).

Tests:
  • test_superseded_by_roundtrips — set on write, reads back via get/recall (AC3/AC6)
  • test_premigration_db_loads_unchanged — open a DB built without the column; rows read with superseded_by=None, no error (AC6)

Gate: pytest tests/test_memory_reconcile.py -k superseded or migration — GREEN

Step 3 — Auto-resolve at write (close the gap) M

Rationale: The core fix. Inside write()'s G-10 block (store.py:542–563), when a rival is found: load it via _row_to_entry, call _pick_winner(entry, rival), quarantine the loser + set superseded_by=winner.id, and write the conflict row already-resolved (resolved_a/_b) — never 'open'. Stays inside the existing try/except: pass so it can never break a write.

Files: store.py write(); tests.

Tests:
  • test_two_contradictions_recall_returns_one — write A then contradicting B (same entity+boundary, same 40-char prefix) → recall() returns exactly one = the winner (AC1)
  • test_loser_quarantined_and_linked — loser has quarantined=1 and superseded_by=winner.id; conflict row not 'open' (AC3)
  • test_exact_dup_still_reinforces_no_conflict — identical content → SM-2 reinforce path (535–539), no conflict row, no quarantine (AC4, regression)
  • test_resolver_failure_does_not_break_write — force _pick_winner to raise → write still succeeds, entry persisted (fail-soft)

Gate: pytest tests/test_memory_reconcile.py + tests/test_memory.py — GREEN

Step 4 — Backlog curator reconcile_open_conflicts() + golden-question rollback M

Rationale: Step 3 only heals new writes; existing status='open' rows (and any pre-fix near-dups) need a one-shot pass. Wrap it in the safety net so a bad demotion is auto-reverted.

Files: store.py — reconcile_open_conflicts() + private _golden_snapshot(top_n) helper (auto-derive golden set from top-N most-accessed entities, OQ-3); tests.

Logic: snapshot recall() for top-N entities → resolve every open row via _pick_winner → re-recall → if any entity's answer set degraded (lost a previously-served live head with no replacement), un-quarantine that pair and flag the row for human review. Returns {resolved, rolled_back, flagged}.

Tests:
  • test_reconcile_clears_open_backlog — seed N open conflicts → all become resolved, losers quarantined+linked (AC5)
  • test_golden_rollback_on_degraded_recall — construct a case where reconciliation would empty an entity's recall → that entity is rolled back + flagged, not silently demoted (AC5)
  • test_reconcile_is_idempotent — second run resolves nothing new, no double-quarantine

Gate: pytest tests/test_memory_reconcile.py -k reconcile or golden — GREEN

Tests:

Rollback Plan

Revert the write() edit → new contradictions log 'open' again (pre-fix behavior); the dormant superseded_by column is harmless. To restore demoted entries: UPDATE memory_entries SET quarantined=0, superseded_by=NULL WHERE superseded_by IS NOT NULL. No data destroyed — fully lossless.
