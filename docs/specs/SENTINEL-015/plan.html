Schema: add MemoryConfig.episodic_recall + BackendConfig.max_turns/max_retries
src/sentinel/config/schema.py

Add 3 fields to MemoryConfig: episodic_recall: bool = True, episodic_recall_top_k: int = 3.
Add 3 fields to BackendConfig: max_turns: int = 30, max_retries: int = 3, base_retry_delay_s: float = 1.0.

  • All 6 fields have safe defaults — existing configs that don't specify them get default behavior.
  • Pydantic validators ensure ge=1 bounds on int fields.
Risk: None. Additive fields with defaults. All pre-015 SentinelConfig round-trips unchanged.
2
RunStore.recall_episodes() — episodic retrieval method
src/sentinel/memory/store.py

Add recall_episodes(target, *, top_k=3, mode=None) → list[RunRecord] to RunStore.
Two-phase: (1) exact entity match, (2) LIKE keyword search on finding_texts. Dedup by entity. Fail-soft.

  • Returns [] when no records or on any exception.
  • Exact match takes precedence over keyword match.
  • Keywords shorter than 4 chars are excluded (stop-word proxy).
  • No new SQL tables or indexes required.
Risk: None. Read-only. The query pattern (LIKE on TEXT) is already used in existing test code.
3
_render_episodic_context() + extend _recall_memory()
src/sentinel/agent/orchestrator.py

Add _render_episodic_context(episodes) → str. Returns "" when episodes is empty (AC-10 parity).
Extend _recall_memory() to also call RunStore.recall_episodes() and append episodic context.
Store episode/entity counts on cfg (temp) for trace emission in run_async().

  • When cfg.memory.episodic_recall = False, episodes = [] and context is "".
  • Episodic block has its own heading and a "do not present as freshly found" instruction.
  • Top 5 finding snippets per episode (150 chars each).
Risk: Low. The memory_context string appended to synthesizer already works; episodic block is an additional append.
4
run_step() — add max_turns + retry policy
src/sentinel/agent/orchestrator.py

Add max_turns: int | None = None, max_retries: int = 3, base_retry_delay_s: float = 1.0 params to run_step().
Pass max_llm_calls=turns to RunConfig. Wrap the body in a retry loop with exponential backoff.
Add _configured_max_turns() helper (reads from global config, mirrors _configured_max_concurrency).

  • On success, returns on first attempt (zero overhead).
  • Retry delay is skipped in tests (pass base_retry_delay_s=0.0 in test fixtures).
  • Trace logs each failed attempt with delay before re-raise.
  • All callers of run_step() continue to work — new params have defaults.
Risk: Medium. RunConfig.max_llm_calls must exist on the installed ADK version. If it doesn't, the field is passed as an unknown kwarg. Mitigation: wrap in try/except TypeError and fall back to RunConfig(streaming_mode=streaming).

run_async() — wire trace entry + pass max_turns/max_retries to run_step
src/sentinel/agent/orchestrator.py

After _recall_memory(), emit the trace line "memory: N entity facts + M episodes recalled".
Pass max_turns=cfg.backend.max_turns, max_retries=cfg.backend.max_retries through _execute_pipeline to run_step.

  • Trace format consistent with existing trace entries.
  • No change to RunResult shape.
Risk: None. The trace is additive.

governance.py — search provider auto-upgrade DDG→Brave
src/sentinel/agent/governance.py

In effective_search_provider(), after resolving the provider: if provider is duckduckgo and BRAVE_API_KEY env var is non-empty, return brave instead.

  • Only upgrades DDG → Brave. Any other provider is untouched.
  • Disabled by setting search.provider: searxng (or any non-duckduckgo value).
  • No key validation in governance — web_search._brave() handles missing-key gracefully.
Risk: None. The Brave backend already exists in web_search.py and is tested.
7
dag.py — inject episodic context into run_dag base_seed
src/sentinel/agent/dag.py

At the top of run_dag(): if cfg.memory.episodic_recall, call RunStore.recall_episodes(target) and append the rendered context to memory_context before passing to _execute_plan.

  • Fail-soft: any exception in the recall is swallowed, memory_context stays unchanged.
  • Works for all domain specialist runs (software/finance/academic/nutrition/travel).
Risk: None. The base_seed merge is additive.
