SENTINEL-015 · Design

1. Architecture Overview

Task start (run_async / run_dag)
        │
        ├─► [EXISTING] Entity memory recall
        │     └─ MemoryStore.recall(target, boundaries) → MemoryEntry[]
        │         └─ formatted as "## Prior memory..." block
        │
        ├─► [NEW] Episodic memory recall (FR-01..05)
        │     └─ RunStore.recall_episodes(target, top_k=3) → RunRecord[]
        │         └─ keyword LIKE search on finding_texts JSON
        │         └─ exact entity match

        │
        └─► Combined memory_context injected into agent seed state
              │
              └─► ADK runner

                    ├─► [NEW] max_turns guard (FR-06)
                    │     └─ RunConfig.max_llm_calls = cfg.backend.max_turns
                    │
                    └─► [NEW] Retry policy (FR-07)
                          └─ 3 retries, exponential backoff (1s/2s/4s)

2. Component Designs

2.1 RunStore.recall_episodes() — new method

Location: src/sentinel/memory/store.py in class RunStore

def recall_episodes(
    self,
    target: str,
    *,
    top_k: int = 3,
    mode: str | None = None,
) -> list[RunRecord]:
    """Top-K episodic recall via keyword search over finding_texts + exact entity match.

    Strategy (priority order):
    1. Exact entity match (same normalized target, any prior run) — most relevant
    2. Keyword LIKE search on finding_texts JSON for non-trivial words from target
       - words < 4 chars are stripped (stop-word proxy)
       - each keyword becomes a LIKE '%word%' condition (OR-joined)
       - results deduped by entity (each entity appears at most once, newest first)

    Never raises: any storage error → return []. Fail-soft as per NFR-03.
    """
    entity = normalize_entity(target)
    # Simple stop-word filter: skip very short words
    keywords = [w for w in entity.split() if len(w) >= 4]

    seen: set[str] = set()
    results: list[RunRecord] = []

    try:
        with _connect(self.path) as conn:
            # 1. Exact entity match — always take ALL prior runs for this target
            exact = conn.execute(
                "SELECT * FROM run_records WHERE entity=? ORDER BY created_at DESC",
                (entity,),
            ).fetchall()
            for r in exact:
                rec = _row_to_run(r)
                if rec.entity not in seen:
                    seen.add(rec.entity)
                    results.append(rec)
                if len(results) >= top_k:
                    return results

            # 2. Keyword LIKE search for related prior runs
            if keywords:
                like_conds = " OR ".join("finding_texts LIKE ?" for _ in keywords)
                params: list = [f"%{k}%" for k in keywords]
                sql = f"SELECT * FROM run_records WHERE ({like_conds})"
                if mode:
                    sql += " AND mode=?"
                    params.append(mode)
                sql += " ORDER BY created_at DESC LIMIT ?"
                params.append(top_k * 5)  # over-fetch to allow dedup

                for r in conn.execute(sql, params).fetchall():
                    rec = _row_to_run(r)
                    if rec.entity not in seen:
                        seen.add(rec.entity)
                        results.append(rec)
                    if len(results) >= top_k:
                        break
    except Exception:
        return []

    return results[:top_k]
Why LIKE, not FTS5? SQLite 3.46.1 ships with FTS5, but a content-FTS5 table requires triggers or explicit re-indexing to stay in sync with run_records. The overhead isn't justified for <10k rows. LIKE on a 10k-row table runs in <5ms. If the dataset grows past 100k rows, migrate to FTS5 at that point.

2.2 _render_episodic_context() — new function

Location: src/sentinel/agent/orchestrator.py, alongside _render_memory_context

def _render_episodic_context(episodes: list[RunRecord]) -> str:
    """Compact episodic block appended to memory_context (SENTINEL-015).

    Returns "" for no episodes → byte-identical to pre-015 (AC-10 parity, FR-04).
    Carries its own leading newlines so it appends cleanly after entity memory.
    """
    if not episodes:
        return ""
    lines = ["\n\n## Episodic Memory: Prior Research Sessions (recalled from history)"]
    lines.append(
        "The following are PAST research sessions. Use as background context. "
        "Verify with fresh research; never present recalled facts as newly found."
    )
    for ep in episodes:
        date = ep.created_at.strftime("%Y-%m-%d") if hasattr(ep.created_at, "strftime") else "?"
        lines.append(
            f"\n### [{date}] {ep.mode} / {ep.target} "
            f"({ep.public} public findings, {ep.gaps} gaps)"
        )
        for txt in ep.finding_texts[:5]:
            snippet = (txt or "")[:150].replace("\n", " ").strip()
            if snippet:
                lines.append(f"  · {snippet}")
    return "\n".join(lines)

2.3 _recall_memory() — extended

Location: src/sentinel/agent/orchestrator.py — extend the existing function.

def _recall_memory(target: str, mode: Mode, cfg) -> tuple[str, object | None]:
    """Fetch prior run (delta) + entity memory + episodic memory (SENTINEL-015).

    Returns (combined_memory_context, prior_run_or_None). Fail-soft: storage errors
    degrade to ("", None) and never break a run. The returned string is empty when
    both entity_memory and episodic_recall are disabled (AC-10 parity).
    """
    prior = None
    try:
        from sentinel.memory import MemoryStore, RunStore
        from sentinel.memory.schema import RunRecord  # noqa: F401 (type import)

        store = RunStore()
        prior = store.latest_for(target)

        # Entity memory (pre-015, unchanged)
        entity_ctx = ""
        if getattr(cfg.memory, "entity_memory", True):
            recalled = MemoryStore().recall(target, allowed_boundaries(mode))
            entity_ctx = _render_memory_context(recalled)
            entity_count = len(recalled)
        else:
            entity_count = 0

        # Episodic memory (SENTINEL-015, FR-01..05)
        episodes: list = []
        if getattr(cfg.memory, "episodic_recall", True):
            episodes = store.recall_episodes(target, top_k=3)

        episodic_ctx = _render_episodic_context(episodes)

        # Single trace token (see FR-05) — logged in run_async below
        cfg._episode_count = len(episodes)   # hack-free alternative: return as 3rd item
        cfg._entity_count = entity_count

        return entity_ctx + episodic_ctx, prior
    except Exception:
        return "", prior
Design note: We stash _episode_count / _entity_count on cfg as a lightweight out-param rather than changing the return type (which would require updating every caller). If this becomes a pattern, refactor to a RecallResult dataclass. For now the stash is an acceptable 2-line trade-off.

2.5 BackendConfig additions (FR-06, FR-07)

Location: src/sentinel/config/schema.py, in class BackendConfig

# Turn controller (SENTINEL-015 FR-06): max LLM calls per step run.
# Passed to ADK RunConfig.max_llm_calls — an agent that loops endlessly
# will be hard-stopped at this count. 30 is generous for a research agent;
# tune down for fast retrieval-only runs.
max_turns: int = Field(default=30, ge=1)

# Retry policy (SENTINEL-015 FR-07): how many times to retry on transient failure.
# Exponential backoff: delay = base_retry_delay_s * (2 ** attempt). Total max
# wait = 1 + 2 + 4 = 7 seconds for max_retries=3. Set to 1 to disable retry.
max_retries: int = Field(default=3, ge=1)
base_retry_delay_s: float = Field(default=1.0, gt=0)

2.6 run_step() — turn controller + retry (FR-06, FR-07)

Location: src/sentinel/agent/orchestrator.py

async def run_step(
    agent, *, message_text, seed_state, streaming, trace,
    max_concurrency: int | None = None,
    max_turns: int | None = None,
    max_retries: int = 3,
    base_retry_delay_s: float = 1.0,
) -> dict:
    """Run one built agent with turn guard + exponential-backoff retry (SENTINEL-015).

    Turn guard: max_turns → RunConfig.max_llm_calls. None = use global config default.
    Retry: on any exception, sleep (1s, 2s, 4s...) and retry up to max_retries times.
    The last attempt re-raises if still failing (fail-loud after exhaustion).
    """
    limit = max_concurrency if max_concurrency is not None else _configured_max_concurrency()
    turns = max_turns if max_turns is not None else _configured_max_turns()

    last_exc: Exception | None = None
    for attempt in range(max(1, max_retries)):
        try:
            async with _leaf_semaphore(limit):
                runner = InMemoryRunner(agent=agent, app_name=APP_NAME)
                session = await runner.session_service.create_session(
                    app_name=APP_NAME, user_id=_USER_ID, state=dict(seed_state),
                )
                for sub in getattr(agent, "sub_agents", []) or []:
                    trace.append(f"agent {sub.name} model={_model_label(sub)}")
                message = types.Content(role="user", parts=[types.Part(text=message_text)])
                run_config = RunConfig(streaming_mode=streaming, max_llm_calls=turns)
                async for event in runner.run_async(
                    user_id=_USER_ID, session_id=session.id,
                    new_message=message, run_config=run_config,
                ):
                    if getattr(event, "partial", False):
                        continue
                    author = getattr(event, "author", "?")
                    if getattr(event, "content", None) and event.content.parts:
                        for p in event.content.parts:
                            if getattr(p, "function_call", None):
                                trace.append(f"{author} → tool:{p.function_call.name}")
                            elif getattr(p, "text", None) and p.text.strip():
                                trace.append(f"{author} · {p.text.strip()[:80]}")
                final = await runner.session_service.get_session(
                    app_name=APP_NAME, user_id=_USER_ID, session_id=session.id
                )
                return dict(final.state)
        except Exception as exc:
            last_exc = exc
            if attempt < max_retries - 1:
                delay = base_retry_delay_s * (2 ** attempt)
                trace.append(
                    f"run_step attempt {attempt+1}/{max_retries} failed "
                    f"({type(exc).__name__}); retrying in {delay:.1f}s"
                )
                await asyncio.sleep(delay)
            else:
                trace.append(f"run_step: all {max_retries} attempts failed; propagating")
    raise last_exc  # type: ignore[misc]

2.8 Search provider auto-upgrade (FR-09)

Location: src/sentinel/agent/governance.py

def effective_search_provider(cfg, *, allow_cloud: bool) -> str:
    """Return the effective search provider.

    SENTINEL-015: if provider is 'duckduckgo' and BRAVE_API_KEY is set,
    auto-upgrade to 'brave' (DDG lite is currently anti-bot blocked).
    """
    provider = cfg.search.provider if not allow_cloud or cfg.search.provider != "gemini" \
               else cfg.search.onprem_fallback
    # auto-upgrade DDG → Brave when key is available (FR-09)
    if provider == "duckduckgo" and os.getenv("BRAVE_API_KEY", "").strip():
        return "brave"
    return provider

5. Key Design Decisions

DecisionChosenRejectedReason
Episodic retrieval mechanism SQLite LIKE on finding_texts FTS5 virtual table, Qdrant Zero new dependencies; 10k rows = <5ms LIKE query. FTS5 needs sync triggers. Qdrant needs a service.
Where to inject episodic context Appended to memory_context string New episodic_context seed key Reuses the existing synthesizer instruction path. No changes to agent prompts needed.
Retry placement run_step (leaf executor) _execute_pipeline (outer loop) run_step is the single leaf every step uses. Retrying there covers DAG steps, coordinator, and legacy pipeline. Outer retry would re-run Pass-1 on a Pass-2 failure.
max_turns parameter RunConfig.max_llm_calls (ADK native) Manual turn counter in event loop ADK's RunConfig.max_llm_calls is the proper hook — it enforces at the framework level and is already in the data path.
Search auto-upgrade In effective_search_provider() At startup / settings UI Governance.py is the single choke-point for provider resolution. The upgrade is invisible to callers and can be disabled by setting provider explicitly to anything other than duckduckgo.
