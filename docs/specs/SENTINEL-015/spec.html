1. Business Problem

Sentinel is a sovereign research agent that runs excellent single-session tasks — but it forgets everything after each session. Every research run starts cold, even if Sentinel has already deeply researched the same target, a related topic, or the same domain last week. The agent is also fragile: DDG lite is anti-botted (0 search results), there is no retry policy on 26B vLLM 502s, and no loop guard against runaway agents.

The user's pilot demo is in 48 hours. The Google AI Challenge judges will look for: agents that learn over time, production-quality resilience, and substantive research output. Sentinel currently lacks all three.

Root diagnosis: The data is already there. run_records (episodic log) and memory_entries (entity facts) both exist in SQLite. They're just never read back to prime new tasks. The biggest single upgrade is injecting existing data — not building new infrastructure.

3. Gap Analysis: Sentinel vs. Production Agent Standard

CRITICAL
Episodic memory is never injected. run_records.finding_texts stores every research output but nothing reads it back to prime the next run. Agent starts cold every time.
CRITICAL
Live web search broken. DDG lite returns HTTP 202 anti-bot challenge (0 results). All research uses parametric model knowledge only — no citations from current web.
HIGH
No retry policy. A single 502 from the 26B vLLM endpoint fails the entire run. The 26B endpoint is known-unstable (SENTINEL streaming-524 notes).
HIGH
No turn controller. A misbehaving tool-call loop can run forever. No max_llm_calls guard on ADK RunConfig.
MEDIUM
Semantic memory (entity facts) injected but not episodic context. Entity MemoryEntry objects are recalled and rendered into synthesizer context. But prior full research sessions (richer context) are not.
MEDIUM
No procedural memory / skill library. Successful research patterns are not captured as reusable skills. Post-deadline.
POST-DEADLINE
No Qdrant / BM25F hybrid retrieval. Full Manthan retrieval stack (Qdrant + pg_search + BGE reranker) is months of work.
POST-DEADLINE
No MCP server. Sentinel research store not exposed to external tools. Adapter contracts not built.

4. Functional Requirements

IDPriorityRequirement
FR-01 MUST When a new task starts, Sentinel SHALL query run_records for past research sessions relevant to the same target or domain and inject a compact summary into the agent's context before the planner runs.
FR-02 MUST The recall method SHALL return up to 3 RunRecord objects. It SHALL match: (1) exact entity name, (2) keyword overlap in finding_texts. It SHALL never raise — degrade to empty list on any storage error.
FR-03 MUST The episodic context block SHALL be clearly labeled "Episodic Memory: Prior Research Sessions" and SHALL instruct the agent to treat recalled facts as background, not as freshly verified.
FR-04 MUST Episodic recall SHALL be gated by cfg.memory.episodic_recall: bool = True. When off, injection is skipped and the run is byte-identical to pre-015 (AC-10 parity).
FR-05 MUST The number of episodes recalled SHALL be recorded in the run trace as memory: N entity facts + M episodes recalled.
FR-06 MUST A max_turns: int = 30 field SHALL be added to BackendConfig and passed to ADK RunConfig.max_llm_calls in run_step, enforcing a hard loop guard on every agent run.
FR-07 MUST A retry policy SHALL be added to run_step: on exception, retry up to 3 times with exponential backoff (1s, 2s, 4s). The number of retries SHALL be configurable via BackendConfig.max_retries: int = 3.
FR-08 MUST Episodic injection SHALL also apply to DAG-routed runs (run_dag / _execute_plan), not only to the legacy competitor/client sequential runs.
FR-09 SHOULD If BRAVE_API_KEY is set in the environment at startup AND search.provider is duckduckgo, Sentinel SHOULD auto-upgrade to brave with a startup log message. This is the web-search fix.
FR-10 SHOULD The run result card in the web UI SHOULD display a "📚 N sessions recalled" badge when episodic memory was injected, giving the user visibility into the memory system.
FR-11 MAY After each successful run, Sentinel MAY extract "what search queries produced citations" and store them as procedural memory entries for future skill reuse. Deferred if time-constrained.

5. Non-Functional Requirements

IDRequirement
NFR-01Episodic recall adds < 50ms to task start time (SQLite LIKE query on < 10k rows).
NFR-02Retry policy adds at most 7 seconds total delay (1+2+4) before propagating a hard failure. The run still fails cleanly if all retries are exhausted.
NFR-03All new code is fail-soft: any exception in episodic recall, retry logic, or turn guard degrades gracefully — it never breaks a completed run.
NFR-04No new external dependencies (no Qdrant, no Redis, no pg_search). SQLite FTS5 (already bundled in Python's sqlite3 3.46.1) is acceptable if needed.
NFR-05Test coverage: every new method has ≥ 1 unit test. The 513 existing tests must still pass.
NFR-06No PII in logs or traces (boundary invariant inherited unchanged).
NFR-07Sovereignty maintained: episodic recall reads only from local SQLite (never cloud). On-prem deployments are not impacted.

6. Out of Scope

  • Qdrant vector store / BM25F hybrid retrieval (post-deadline)
  • Full Manthan adapter contracts (Auth/Identity/Knowledge/Action/Audit)
  • MCP server surface exposing Sentinel research store
  • Procedural memory / skill library (FR-11 is MAY)
  • Context window manager / summarization (no clear need at current scale)
  • Sandboxed execution
  • Telemetry / Prometheus metrics
  • Cross-project shared memory (single-operator scope)

7. Acceptance Criteria

  1. Running a Razorpay finance research task for the SECOND time injects at least 1 prior episode into the agent context. Visible in the run trace as memory: ... + M episodes recalled.
  2. RunStore.recall_episodes("Razorpay") returns at least 1 record after the first run has completed.
  3. A simulated ADK runner failure retries 3 times before propagating. Verifiable via unit test with a mock runner that fails N times.
  4. BackendConfig.max_turns = 5 causes the runner to stop after 5 LLM calls (unit test with mock runner).
  5. Setting cfg.memory.episodic_recall = False produces a run with no episodic context block (byte-identical to pre-015 for the memory_context string).
  6. All 513 pre-existing tests continue to pass.
  7. At least 8 new tests cover the new functionality.
